# Logfire Dashboard Backups

Local version-controlled snapshots of Logfire dashboards (Perses-compatible JSON).

## Why

Logfire dashboards are edited through the UI and don't have built-in version history outside Logfire's own `version` counter. Keeping a snapshot here gives us:

- A restorable backup if a dashboard is accidentally broken.
- Reviewable diffs when someone changes a panel.
- SQL extracted into `.sql` files so queries are editable with syntax highlighting instead of one-line escaped JSON.

## Layout

One folder per dashboard. Within each folder:

```
llm-cost/
├── template.json        human-authored; query strings are replaced by {"$file": "queries/foo.sql"} refs
├── queries/*.sql        each panel's SQL, diffable and editable
└── rendered.json        generated by render.py; this is what gets pushed to Logfire
```

`render.py` (in this folder) walks `template.json`, replaces every `{"$file": "<path>"}` with that file's contents as a string, and writes `rendered.json`. No other template syntax is supported.

## Common operations

### Edit a query
1. Edit the `.sql` file in `queries/`.
2. `python logfire/dashboards/render.py <dashboard-name>` to regenerate `rendered.json`.
3. Commit both the `.sql` change and the regenerated `rendered.json`.
4. Push `rendered.json` to Logfire (see below).

### Refresh from Logfire (someone edited in the UI)
1. Fetch the current definition: `mcp__logfire__dashboard_get(dashboard="<slug>", project="portfolio")`.
2. Write it to `<dashboard-name>/rendered.json`.
3. Update `template.json` and `queries/*.sql` to match any SQL or panel changes.
4. `python logfire/dashboards/render.py --check` must exit 0 before committing.

### Push local changes up to Logfire
```
mcp__logfire__dashboard_update(
  dashboard="<slug>",
  project="portfolio",
  definition=<contents of <dashboard-name>/rendered.json>,
)
```

### CI / pre-commit integrity check
`python logfire/dashboards/render.py --check` exits non-zero if any `rendered.json` has drifted from its `template.json` + SQL. Use this to catch hand-edits to `rendered.json` that weren't propagated back to the template.

## Adding a new dashboard

1. `mkdir logfire/dashboards/<name>/queries`
2. Write `<name>/template.json` (copy another dashboard's as a starting point).
3. Put each panel's SQL in `<name>/queries/<query>.sql`.
4. Reference them from the template: `"query": { "$file": "queries/<query>.sql" }`.
5. `python logfire/dashboards/render.py <name>` to produce `rendered.json`.
6. Push to Logfire via `dashboard_update`.
